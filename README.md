## Key Features

- A Hybrid systems seems to go for analyzing a share. Thats how a framework will be designed from now on (2025-09-26).
    - This would reduce the number of prompts for share analysis and backtesting *significantly*.

- [2025-10-15] A hybrid system means a combination of a financials engine for calculating & analyzing hard numbers - and an llm engine for textual analysis.
- [2025-10-15] Within the LLM engine, there would be 2 subparts
    - **Investor persona profiling** - Generating relevant context, or fine-tuning a model to mimic an investor's behavious or decision making pattern. A kind of an advanced summarization of principles/ frameworks/ rules/ guidelines for investing.
    - **Correlation engine** - Using the rules generated by IPP as context, to instruct the LLM to correlate the available share data with those rules.


## Components
So obviously, a bunch of componenents, or 'agents' would provide signals to take various actions. 
The scope of these actions can range from individual shares to portfolio or market as a whole. For example:
- Entry & Exit signals for specific stocks 
- Share rankings
- Scoring/ Eval Framework

A multi agent system (MAS).


## Potential Problems
List of potential problems:
- Lack of relevant data.
- Lack of historical data for backtesting.
- Highly unstructered data.
